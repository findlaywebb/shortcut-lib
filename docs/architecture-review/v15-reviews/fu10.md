## 4. Assessment Against the Brief

### Factory signatures

| Method | Body positional | `*` enforced | `body_type` | `method` default |
|--------|----------------|-------------|-------------|-----------------|
| `.get(url, *, headers)` | None | Yes | Not set | Not applicable |
| `.json(url, body, *, method, headers)` | `dict[str, Any]` | Yes | `"JSON"` | `"POST"` |
| `.plain_text(url, body, *, method, headers)` | `ParamValue` | Yes | `"Plain Text"` | `"POST"` |
| `.file(url, body, *, method, headers)` | `ParamValue` | Yes | `"File"` | `"POST"` |

All four match the brief exactly. `method` and `headers` are keyword-only on every body-accepting factory. The `Literal["POST", "PUT", "PATCH", "DELETE"]` annotation on `method` narrows the type surface correctly — `ty` validates this statically (passes).

### `.get()` body isolation

Confirmed: `.get()` accepts only `url` (positional) and `headers` (keyword-only). Passing `**{"body": NamedVar("X")}` raises `TypeError: DownloadURL.get() got an unexpected keyword argument 'body'` at runtime — no workaround via `**kwargs` spreading. The test at line 476 (`test_get_factory_no_body_kwarg`) covers this and asserts `TypeError` not `SchemaError`, which is correct.

### `body_type` routing

- `.json()` passes `body_type="JSON"` → `_params` routes to `WFJSONValues` via `_DICT_BODY_TYPES`.
- `.plain_text()` passes `body_type="Plain Text"` → `_params` routes to `WFRequestVariable`.
- `.file()` passes `body_type="File"` → same `WFRequestVariable` path.
- Non-dict body passed to `.json()` at runtime hits the existing guard in `_params` (line ~220): raises `SchemaError` naming the type. The static type annotation (`body: dict[str, Any]`) additionally makes `ty` flag this at check time.

### GET wire format

`DownloadURL.get("https://x")` emits no `WFHTTPMethod`, no `WFHTTPBodyType`, no `WFJSONValues`, no `WFRequestVariable`. Verified by `test_get_factory` (line 415).

### Headers

Passed through as `headers=headers` in all four factories; empty/`None` case correctly omits `ShowHeaders` and `WFHTTPHeaders` (pre-existing `if self.headers:` guard in `_params`).

### Docstring style

The class docstring now leads with a four-line factory example block before the `Args:` section, mirroring the `AskForInput` pattern on `main` exactly (same structure: intro sentence, code block, then direct-constructor paragraph). Verified against `e1bac0f:src/shortcut_lib/schema/actions/ask.py`.

The per-method docstrings follow Google style — imperative summary ≤ 72 chars, `Args:` / `Returns:` sections, no trailing noise.
