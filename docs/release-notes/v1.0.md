# v1.0 — Release Notes (draft)

**Date:** 2026-05-09
**Status:** V1 complete. v1.0 tag and repo-public flip are the user's call.

V1 is the first stable release. shortcut-lib authors and signs Apple Shortcuts
files programmatically. Its primary author is Claude Code — the lib is what the
LLM calls into, not what calls the LLM.

---

## Highlights

- **24 leaf actions + 4 control-flow constructs.** Tier 0/1/2 + Apple
  Intelligence (`UseModel`, Writing Tools). Auto-discovering registry; `RawAction`
  passthrough for the 365 unmodelled action identifiers in the corpus.

- **Wire-format equivalence sweep.** 336 tests total, including a deliberate
  sweep over 24 leaf actions + 4 control-flow constructs confirming the emitted
  dicts match Apple GUI emission byte-for-byte where sampling allows. The
  sweep surfaced four schema bugs (three envelope-shape bugs + one `ChooseFromMenu`
  emit flaw), all fixed before V1 closed.

- **Envelope-shape oracle.** `scripts/scan_envelope_types.py` maps every action
  parameter slot to its observed envelope shape across all decoded samples and
  writes `data/observed_envelope_types.json` as a CI artefact. `coerce_text_field`
  in `schema/base.py` rewraps bare attachments as `WFTextTokenString` for every
  slot where Apple requires that envelope — closing the class of bug that caused
  silent iOS failures during on-device validation.

- **Setup-section authoring (FU-9).** `Shortcut.ask_on_import` and
  `Shortcut.ask_text_on_import` emit `WFWorkflowImportQuestions` entries —
  the "drag in, fill form, run" UX that makes shared shortcuts safe to
  distribute without baking credentials into the file.

- **Four real-target shortcuts shipped.** See "Validated targets" below.

- **Composition story settled.** iOS assigns fresh UUIDs at import time and
  ignores any identifier baked into a locally-signed `.shortcut` file.
  Composition is therefore in Python (helper functions building one workflow),
  not `RunWorkflow` (which remains available for cases with a genuinely separate
  trigger). This pivot is documented in `docs/roadmap.md` decisions log and
  `examples/VALIDATION_vault_note_to_git.md`.

- **Typed variable handles.** `Shortcut.set` returns a `NamedVar[T]` generic
  handle; downstream actions receive a typed variable reference rather than a
  bare string, catching misspelled variable names before the build runs.

- **Factory methods for `AskForInput`.** `AskForInput.number(...)` and
  `AskForInput.text(...)` surface the right dependent fields at the call site;
  `__post_init__` validation gives the LLM a clear error message when
  constraints are violated.

## Architecture decisions worth knowing

**The lib is what LLMs call into, not what calls LLMs.** The natural-language
compiler direction (LLM → Python AST → lib calls) is not pursued in V1.
shortcut-lib is a typed surface for Claude Code (or any Python author) to call
directly. The LLM uses `describe_action`, the registry, and the error messages
as its interface.

**Composition is in Python, one shortcut per build.** The original brief called
for compositional helper shortcuts linked via `RunWorkflow`. iOS strips and
reassigns shortcut UUIDs at import time; a locally-signed `.shortcut` file
cannot pre-link helpers — every import requires manual re-selection. The
composition story is therefore Python helper functions building a single
self-contained workflow. `RunWorkflow` remains in the schema for cases with a
genuinely separate trigger (Setup-only auth helpers, iCloud-shared utilities).

**Setup section (FU-9) for credentials.** Every V1 shortcut collects its PAT
and repo via `WFWorkflowImportQuestions`. This is the only safe distribution
pattern for shortcuts that call external APIs. Never bake a real PAT into a
signed `.shortcut` file.

**Envelope oracle as empirical wire-format source of truth.** `data/observed_envelope_types.json`
is the canonical record of which parameter slots use `WFTextTokenString` vs
`WFTextTokenAttachment` vs bare scalars. When Apple's behaviour diverges from
what a new sample shows, the oracle is updated and `coerce_text_field` is
adjusted accordingly.

## Known limitations

**Three xfailed equivalence tests (FU-13).** In `tests/test_wire_format_equivalence.py`:

- `test_repeat_count_wire_format` — the corpus only has a demo placeholder for
  `RepeatCount` with a configured non-default count; needs a fresh sample.
- `test_choose_from_menu_wire_format` — the available sample (`read_later.xml`)
  was authored with an older Shortcuts editor; needs a fresh sample from current
  Shortcuts.app.
- `test_text_split_wire_format` — the schema does not yet model the
  `Show-text` UI toggle that one sample carries.

**GitHub create-or-update sidestep (FU-8).** The vault-note and quick-task
shortcuts avoid GitHub's "must supply sha when file exists" requirement by using
millisecond-precision filenames (`yyyy-MM-dd_HH-mm-ss-SSS`). A robust path
needs GET-then-PUT-with-sha, which requires a JSON-extract action not yet
modelled. Deferred to V1.5.

**365 unmodelled action identifiers.** The corpus contains 393 distinct action
identifiers; 24 are modelled with typed schemas. `RawAction` passthrough handles
the rest — any unrecognised identifier round-trips correctly but cannot be
authored via the typed API.

**iOS 26 sample coverage gap.** Wire-format R1 noted iOS 26 added 25+ new
actions (Visual Intelligence, Create Image, Find Message, Search Photos) with
no corpus coverage. These are not modelled in V1.

**Setup-question targeting is top-level only.** `ask_on_import` targets actions
at the top level of the workflow. Targeting actions nested inside `If`, `RepeatEach`,
or `ChooseFromMenu` bodies is V1.5.

## What's deferred to V1.5

Pulled from `docs/handoff.md` follow-up items:

- **MCP server publication** — `shortcuts-mcp` scaffold can be drafted but
  should not go live until the wire-format equivalence sweep is complete and
  production-proven.
- **`DownloadURL` factory methods (FU-10)** — `DownloadURL.json(url, body, ...)`,
  `.plain_text(url, body)`, `.get(url, ...)` to surface the right body-type
  wire keys without caller errors.
- **Validation-engine consolidation (FU-12)** — a single
  `validate_workflow(workflow) -> list[ValidationFinding]` entry point consumed
  by both the MCP server and any future audit CLI. Today validation is split
  across `__post_init__` raises, the envelope oracle, and the equivalence sweep.
- **iOS 26 new-action sample coverage** — add corpus samples for Visual
  Intelligence, Create Image, Find Message, and other iOS 26 additions.
- **Robust create-or-update for GitHub (FU-8)** — GET-then-PUT-with-sha instead
  of the millisecond-filename workaround.
- **`@action` decorator + frozen-by-default** — closes the post-construction
  mutation footgun; requires B7+E1 (`RunWorkflow` self-reference resolution)
  as a prerequisite.
